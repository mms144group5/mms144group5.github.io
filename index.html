Introduction

Review of Common DBMS Terminologies and Concepts

A Database Management System (DBMS) are computer programs, applications or software that manages a database. It handles the storage, retrieval and updating of data. Operators use the system to run different types of operations with the data.


Most DBMS functions involving adding, updating and editing of data. DBMS systems was usually held by large companies, organizations and even the government as they had the resources available to manage large pieces of data. This was in the form of record keeping, accounting, census and survey tracking, human resources, customer support and similar types of systems.


However, with constant innovating technology, DBMS is now available to anyone with a computer and internet access. Requesting and managing data has never been easier. An example of this would be the task of collecting survey data. Before computer systems a person had to go door to door, speak with individual people hand write data on paper to collect information. Imagine how long would have taken to find out what favorite type of food 1000 people would like. Now imagine today, a person could just create a survey on Survey Monkey or Facebook and post the questions on their news feed. They could get survey answers in a few hours or days, which would have taken weeks or months to obtain the data.

Database Management Systems DBMS

Here are some of the basic functions of a DBMS:

  • Adding Records: Accepting stored data from a user
  • Modifying Records: Modifying or updating existing data
  • Deleting Records: User deleting stored data
  • Search Function: This function is mostly used for reviewing, modifying or deleting records
  • Indexing & Cataloging: This facilitates faster functions when data is sorted into groups.

Review of Common DBMS Terminologies and Concepts*

  • Data - known facts that could be recorded and stored on computer media (Figueroa)
  • Database - a structured collection of data
  • Database Management System (DBMS) - a collection of interrelated data plus the software and hardware required to access that data (Alviar, et al. 1998)
  • Relational DBMS - is a collection of organized set of tables from which data can be accessed easily. It is the basis for SQL and for all modern database systems such as MS SQL Server, IBM DB2, Oracle, MySQL, and Microsoft Access.
  • Structured Query Language (SQL) - is a standard language for storing, manipulating and retrieving data in databases.

Review of Common DBMS Terminologies and Concepts

  • Entity - a piece of data to be stored, pertaining to one real-world object, person, place etc.
  • Attribute - smaller piece of information about entities
  • Table - a collection of related data entries. It consists of columns and rows.
  • Record - consists of information that is related one way or another
  • Field - pieces of information that comprise a record

SQL Statements

  • SQL statements are used to perform actions on a database
  • SQL keywords are not case sensitive
  • Some database systems require a semicolon at the end of each SQL statement

Important SQL COMMANDS

  • SELECT - extracts data from a database
  • UPDATE - updates data in a database
  • DELETE - deletes data from a database
  • INSERT INTO - inserts new data into a database
  • CREATE DATABASE - creates a new database
  • ALTER DATABASE - modifies a database
  • CREATE TABLE - creates a new table
  • ALTER TABLE - modifies a table
  • DROP TABLE - deletes a table
  • CREATE INDEX - creates an index (search key)
  • DROP INDEX - deletes an index

Adding Records

Selecting Records

The SELECT statement is used to select data from a database.


The data returned is stored in a result table, called the result-set.


When selecting data, the syntax would be:



SELECT column1, column2, …

FROM table_name;



Here, column1, column2, ... are the field names of the table you want to select data from.


If you want to select all the fields available in the table, use the following syntax:



SELECT * FROM table_name;

ADDING NEW RECORDS

The INSERT TO statement is used to insert/add new records in a table.

When adding values to specific columns of the table, both the column names and values should be specified. We use the SQL statement below:


INSERT INTO table_name (column1, column2, column3, ...)

VALUES (value1, value2, value3, ...);


When adding values to all columns of the table, there is no need to specify the column names in the SQL query. However, the order of values should be the same as the columns in the table. We write a simplified SQL statement:


INSERT INTO table_name

VALUES (value1, value2, value3, ...);

Example: Adding New Records

Information on the airline’s newly hired employees should be added to the database. We do this by specifying which table are we’re including this new record and then adding data on specific or all columns of the table.


Refer to the “Employees” table from the Airline Database. Margaret Rye was recently hired by the airline and we want to add her record in the Employees table of the airline’s database. The following SQL statement inserts a new record in the Employees table:


INSERT INTO Employees (ID, Name, Age, Designation, Status)

VALUES (107, ‘Margaret Rye’, 29, ‘Ground Staff’, ‘Contractual’);

Modifying Records

Modifying or Updating Records

The UPDATE statement is used to modify the existing records in a table. The syntax would be:


UPDATE table_name

SET column1 = value1, column2 = value2, …

WHERE condition;


Important Note!


Be careful when updating records in a table! The WHERE clause specifies which record(s) should be updated. If you omit the WHERE clause, all records in the table will be updated!

Example: Modifying/Updating Records

In this Customer table, say we want to update Olivia Wilde’s email and city because she recently relocated in Manila to study in UP Diliman.
